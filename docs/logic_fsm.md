
# ⚙️ 状态机与核心逻辑规范 (Logic & FSM)

本文档规定环境监控与安防联动的核心判定逻辑，解决自动判定与手动指令的冲突问题。

## 1. 系统工作模式
系统包含以下三个核心状态：
1. `AUTO`: 默认状态，按温湿度阈值自动控制风扇。
2. `MANUAL_ON`: 手动强制开启风扇，优先级最高。
3. `MANUAL_OFF`: 手动强制关闭风扇，优先级最高。

## 2. 状态转移与执行动作条件

### 2.1 散热风扇控制逻辑
* **AUTO 开启阈值**:
  * 温度 `>= 30.0°C`
  * 或湿度 `>= 70.0%`
* **AUTO 关闭恢复阈值**:
  * 温度 `<= 28.0°C`
  * 且湿度 `<= 65.0%`
* **手动模式抢占**:
  * 收到 `on` 后立即切换到 `MANUAL_ON`
  * 收到 `off` 后立即切换到 `MANUAL_OFF`
  * 收到 `auto` 后恢复 `AUTO`
  * 手动模式下忽略温湿度阈值判断

### 2.2 安防报警逻辑
* **触发条件**: HC-SR501 检测到信号跳变，外部中断上升沿触发
* **执行动作**:
  * 立即拉高蜂鸣器输出
  * 持续鸣叫 `5000ms`
  * 在报警窗口内，`/api/status` 的 `intrusion_alert` 置为 `true`
* **报警结束**:
  * 超时后自动关闭蜂鸣器
  * `intrusion_alert` 自动恢复为 `false`
