
# ⚙️ 状态机与核心逻辑规范 (Logic & FSM)

本文档规定了环境监控与安防联动的核心判定逻辑，解决自动判定与手动指令的冲突问题。

## 1. 系统工作模式 (有限状态机)
系统包含以下三个核心状态节点：
1. **AUTO (自动模式)**: 默认状态。
2. **MANUAL_ON (手动强制开启)**: 优先级最高。
3. **MANUAL_OFF (手动强制关闭)**: 优先级最高。

## 2. 状态转移与执行动作条件

### 2.1 散热风扇控制逻辑
* **触发 AUTO 模式开启风扇的阈值**: 
  * 温度 `> [填写报警温度，如 30.0°C]` 
  * **或** 湿度 `> [填写报警湿度，如 70.0%]`
* **退出 AUTO 开启的阈值 (需考虑防抖/回差)**: 
  * 温度 `< [填写恢复温度]` **且** 湿度 `< [填写恢复湿度]`
* **手动模式抢占**: 无论当前温湿度如何，一旦收到 `MANUAL_ON` 或 `MANUAL_OFF` 指令，立刻执行对应动作并锁定状态，直到收到恢复自动模式的指令。

### 2.2 安防报警逻辑
* **触发条件**: 红外传感器检测到信号跳变 (硬件中断触发)。
* **执行动作**: 立刻拉高蜂鸣器电平，持续鸣叫 `[填写鸣叫时长，如 5000ms]`，并在 `/api/status` 中将报警标志位设为 `true`。
